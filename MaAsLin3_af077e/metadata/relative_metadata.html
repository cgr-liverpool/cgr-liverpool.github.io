Relative metadata

In this page we’ll carry out correlation analysis of the metadata from the “Relative” dataset.

Jupyter-notebook

In the “metadata_exploration” create a new R jupyter notebook called “1-Relative_metadata_exploration.ipynb”.

Library

Load the only library we need: variancepartition.

#Library
library("variancePartition")

Data

Load the metadata data.frame we created in the data formatting part of the Relative section. This is the exact format we need for the correlation functions below.

#Load metadata
load("../relative/metadata.RData")

View the top of the metdata.

#View head of data.frame
metadata |> head()

Formula

Create an object containing the formula we will test. In this case we’ll include all the variables present in the metadata to see how they all correlate to each other.

#Create formula that would be used for MaAsLin3 with all 
# variables of interest
form <- ~ SampleSite + TimepointName + Timepoint + Reads + (1|SubjectID)

Pairwise correlations

Now we can create a matrix of all the pairwise correlations between the metadata variables with the canCorPairs() function. This requires the formula (created above) and then the metadata data.frame.

C <- variancePartition::canCorPairs(form, metadata)

We can view the matrix where values of 1 mean complete correlation and 0 means no correlation.

C

Correlation heatmap

The plotCorrMatrix() function allows us to quickly create a correlation heatmap.

#Create correlation heatmap
variancePartition::plotCorrMatrix(C)

Naturally TimepointName and Timepoint completly correlate. More interesting is that Reads has quite high correlation to all the other variables. We can explore this with some tidyverse functions.

Correlation exploration

We’ll look at two of the correlations:

  1. The Reads and SampleSite to look at continuous versus categorical
  2. The SampleSite and SubjectID to look at categorical versus categorical

Reads and SampleSite

The Reads and SampleSite variables have a correlation of 0.338. We can quickly see why with a boxplot.

metadata |>
    ggplot2::ggplot(aes(x = SampleSite, y = Reads)) +
    ggplot2::geom_boxplot()

In this case it looks like the Axilla samples have smaller read counts than the Cheek and Scalp samples. This is a normal difference between different environments and nothing to worry about.

SampleSite and SubjectID

The SampleSite and SubjectID variables have a correlation of 0.11. First we can count the number of each pairing.

#Summarise to see counts of each Sample and Subject pairing
metadata |>
    dplyr::count(SampleSite, SubjectID)

If you look through you may be able to see what is causing the correlation. For a clearer view lets use the function ggplot2::geom_tile().

metadata |>
    dplyr::count(SampleSite, SubjectID) |>
    ggplot2::ggplot(aes(x=SampleSite, y=SubjectID, fill=n)) +
    ggplot2::geom_tile()

With the tile plot it is clear the correlation is caused by the Axilla samples. There are 2 samples from each subject for the Cheek and for Scalp. However, some of the subjects only have 1 Axilla sample, with one subject having none.

I wouldn’t worry about the correlations we found in this metadata. It is very likely that read depth will correlate with other variables and the MaAsLin3 developers advice always including it as a random effect for non-quantitative datasets.

We don’t have 2 continuous variables to compare but if you did you could create a scatter plot to view their collinearity.
